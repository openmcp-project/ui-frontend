# New API Requests

The project has two data-fetching stacks. Choose the right one for the space you are working in.

## Rule: Never Call the Backend Directly from the Client

All requests must go through `/api` on the BFF (`server.ts`). The BFF proxies to the real backend and handles auth. Direct backend calls from the browser will be blocked by CORS and cookie policy.

---

## REST + SWR (MCP space)

Used in `src/spaces/mcp/` and via `src/lib/api/useApiResource.ts`.

### Routing Headers

The BFF uses request headers to route a single `/api` fetch to the correct cluster:

| Header | Purpose |
|---|---|
| `X-project` | Target project |
| `X-workspace` | Target workspace |
| `X-mcp` | Target Managed Control Plane |
| `X-use-crate` | Use crate API instead of MCP API |
| `X-jq` | Server-side jq filter applied before returning the response |

### Adding a New Resource Fetch

1. Define the response type in `src/lib/api/types/` (Zod schema + inferred TypeScript type).
2. Create a hook using `useApiResource` or `useSWR` directly.
3. Pass routing headers via the `ApiConfig` / `fetchApiServer` helpers.

```typescript
import { useApiResource } from '@/lib/api/useApiResource';
import { MyResourceSchema } from '@/lib/api/types/my-resource';

export function useMyResource(project: string, workspace: string) {
  return useApiResource('/my-resource', MyResourceSchema, {
    headers: { 'X-project': project, 'X-workspace': workspace },
  });
}
```

### Mutations

For create / update / delete operations, use the hooks in `src/hooks/`. See `src/hooks/useCreateProject.ts` for a pattern. Invalidate the relevant SWR cache key after a successful mutation.

## GraphQL + Apollo (Onboarding space)

Used in `src/spaces/onboarding/` against `/api/onboarding/graphql`.

### Generated Types

**Do not hand-write GraphQL response types.** All types are generated by `@graphql-codegen/client-preset` into `src/types/__generated__/graphql/`. See the [Generating GraphQL Types](../../README.md#generating-graphql-types) section of the README for how to regenerate them when the schema changes.

The schema is served by the [kubernetes-graphql-gateway](https://github.com/platform-mesh/kubernetes-graphql-gateway).

### Adding a New Query or Mutation

1. Write the `.graphql` file alongside the component or hook that uses it.
2. Ask the user to run codegen to regenerate types (`npm run generate-graphql-types -- <token>`).
3. Import the generated hook from `src/types/__generated__/graphql/`.

---

## State Management

There is no Redux / Zustand / MobX. State lives in:

- **SWR cache** — server state for REST resources
- **Apollo cache** — server state for GraphQL resources
- **React local state / context** — UI-only state

Do not introduce a global state library.
